# v3.0 Operator Runbook — homelab-admin

Operator-first reference for production ops of `homelab-admin` (Next.js 15 + Bun on mcow). No narrative — concrete commands.

Covers: deploy, rollback, secret rotation, Auth.js session reset, Caddy reload, exporter restart, backup/restore, failure modes, DNS/TLS. Phase 22 deliverables D-22-14 / D-22-15 / D-22-16.

---

## 1. Infrastructure Map

- **Target host:** `homelab.makscee.ru` on mcow (Tailnet IP `100.101.0.9`).
- **Ingress:** Caddy — Tailnet-only external surface, TLS via LE HTTP-01. SEC-01 rate limit on `/api/auth/*` (60 req/min/IP, D-22-06).
- **App:** Next.js 15.5.15 + React 19, running on Bun 1.1.38 (pinned — do not bump without P-05 re-validation).
- **Systemd unit:** `homelab-admin.service` (user `homelab-admin`, port `3847` loopback).
- **Audit DB:** `/var/lib/homelab-admin/audit.db` (sqlite, bun:sqlite driver).
- **Secrets:** `secrets/mcow.sops.yaml` (SOPS + age; age key at `/root/.config/sops/age/keys.txt` on mcow, mirror in operator hub).
- **Reference:** `/Users/admin/hub/workspace/homelab/CLAUDE.md` (infrastructure map — authoritative).

Quick host check:
```bash
ssh root@mcow 'hostname && systemctl is-active homelab-admin.service && systemctl is-active caddy'
```

## 2. Deploy Procedure

```bash

ansible-playbook -i inventory/homelab.yml playbooks/deploy-homelab-admin.yml
```

Expected PLAY RECAP:
```

```

Verification (post-deploy):

```bash
curl -s  https://homelab.makscee.ru/api/health | jq .  # { ok: true, ... }
ssh root@mcow 'systemctl status homelab-admin.service' # Active: active (running)
```

## 3. Rollback (D-22-16)

Rollback = redeploy at a known-good commit SHA. No automated rollback tooling. No schema reverse step in v3.0 — migrations are additive only.

One-line form (D-22-16 canonical):
```
ansible-playbook deploy-homelab-admin.yml -e ref=<prev-sha>
```

Full procedure with inventory:
```bash
ansible-playbook -i ansible/inventory/homelab.yml \
  ansible/playbooks/deploy-homelab-admin.yml -e ref=<prev-sha>
```

Concrete example — roll back to last-known-good at time of v3.0 ship:
```bash
ansible-playbook -i ansible/inventory/homelab.yml \
  ansible/playbooks/deploy-homelab-admin.yml -e ref=46ccb76
```

Post-rollback checks:
```bash
curl -s https://homelab.makscee.ru/api/health | jq .ok                   # true
ssh root@mcow 'systemctl status homelab-admin.service | head -5'         # active
ssh root@mcow 'sqlite3 /var/lib/homelab-admin/audit.db "SELECT COUNT(*) FROM audit_log;"'
```

If a rolled-back build reads the live `audit.db` successfully and `/api/health` returns 200 with `ok:true`, rollback is complete.

## 4. Secret Rotation

Secrets file: `secrets/mcow.sops.yaml` (SOPS + age, committed encrypted).

Per-key rotation:
```bash
# Edit decrypted in $EDITOR
sops secrets/mcow.sops.yaml

# Deploy env-only (skips Next.js build)
cd ansible
ansible-playbook -i inventory/homelab.yml playbooks/deploy-homelab-admin.yml --tags env

# Handler "Restart homelab-admin" fires automatically on env change.
```

**GitHub OAuth rotation** (when app credentials compromised or rotated):
1. github.com/settings/developers → OAuth Apps → regenerate client secret (and/or create new app).
2. `sops secrets/mcow.sops.yaml` → update `homelab_admin.AUTH_GITHUB_ID` and `homelab_admin.AUTH_GITHUB_SECRET`.
3. Also rotate `AUTH_SECRET` in the same commit (see section 5) to invalidate live sessions.
4. Callback URL must remain `https://homelab.makscee.ru/api/auth/callback/github`.
5. Deploy: `ansible-playbook ... --tags env`.

## 5. Auth.js Session Reset

Invalidate every live session (breach response, forced re-sign-in):
```bash
# 1. Generate new secret
openssl rand -hex 32

# 2. Edit sops file, replace AUTH_SECRET with output above
sops secrets/mcow.sops.yaml

# 3. Deploy env
cd ansible
ansible-playbook -i inventory/homelab.yml playbooks/deploy-homelab-admin.yml --tags env
```

Effect: all JWT sessions signed by the previous secret become invalid on next request; operator must re-sign-in via GitHub OAuth.

## 8. Backup / Restore Drill (D-22-14)

**Backup (cron nightly 03:17 UTC):**
```bash
ssh root@mcow /usr/local/sbin/backup-audit-db.sh
ssh root@mcow 'ls -la /var/backups/homelab-admin/ | tail -5'
```
Files: `/var/backups/homelab-admin/audit.db.<utc-ts>.gz` — retained 14 days (dir mode 0700 root).

Cron entry (installed by `deploy-homelab-admin.yml`):
```
17 3 * * * /usr/local/sbin/backup-audit-db.sh >> /var/log/homelab-admin-backup.log 2>&1
```

**Restore (drill or recovery):**
```bash
# One-shot drill on mcow (no side-effects — writes to /tmp scratch):
LATEST=$(ssh root@mcow 'ls -1t /var/backups/homelab-admin/*.gz | head -1')
ssh root@mcow "bash /tmp/restore-audit-db.sh '$LATEST'"

# Or from controller with scripts/launch/restore-audit-db.sh checked out:
scp scripts/launch/restore-audit-db.sh root@mcow:/tmp/
ssh root@mcow "chmod +x /tmp/restore-audit-db.sh && /tmp/restore-audit-db.sh '$LATEST'"
```

**Drill evidence — 2026-04-21T15:51:05Z (Phase 22 plan 03 task 1):**
```
LATEST=/var/backups/homelab-admin/audit.db.20260421T155105Z.gz
[restore] live=0 restored=0 scratch=/tmp/homelab-admin-restore-2475342
[restore] PASS — integrity ok; row counts comparable
```
`live=0 / restored=0` reflects an empty `audit_log` table at drill time (fresh v3.0 deploy, no admin actions yet logged). `PRAGMA integrity_check` returned `ok` on the restored DB — backup path + gzip round-trip + sqlite integrity all verified.

**Full recovery (live DB corruption):**
```bash
ssh root@mcow '
  systemctl stop homelab-admin.service
  mv /var/lib/homelab-admin/audit.db /var/lib/homelab-admin/audit.db.broken
  gunzip -c /var/backups/homelab-admin/<chosen>.gz > /var/lib/homelab-admin/audit.db
  chown homelab-admin:homelab-admin /var/lib/homelab-admin/audit.db
  systemctl start homelab-admin.service
'
curl -s https://homelab.makscee.ru/api/health | jq .ok   # expect true
```

## 9. Common Failure Modes

| Symptom | Check | Fix |
|---------|-------|-----|
| **Caddy 502** at `https://homelab.makscee.ru` | `ssh root@mcow 'systemctl status homelab-admin.service'` | bun crash → `journalctl -u homelab-admin -n 200 --no-pager`; usually bad env or missing dep → fix and redeploy. |
| **429 on /api/auth/*** | Expected at >60 req/min from same IP (SEC-01, D-22-06). | Operator wait 60s or check if brute-force; review `journalctl -u caddy` for source IPs. |
| **OAuth 500 on /api/auth/callback** | `ssh root@mcow 'cat /etc/homelab-admin/env \| grep AUTH'` (as root) | Missing/invalid `AUTH_SECRET` or `AUTH_GITHUB_ID/SECRET`. Confirm GitHub app callback URL = `https://homelab.makscee.ru/api/auth/callback/github`. Fix in `secrets/mcow.sops.yaml` → redeploy `--tags env`. |
| **audit.db locked** (5: database is locked) | `ls /tmp/homelab-admin-restore-*` for stale restore scratch dirs. | `ssh root@mcow 'rm -rf /tmp/homelab-admin-restore-*'` then re-check; if still locked, `systemctl restart homelab-admin.service`. |
| **Prometheus scrape failing on homelab-admin** (D-22-17, added in plan 22-06) | `ssh root@mcow 'ss -ltnp \| grep 9300'` | Metrics endpoint not bound — service crashed or misconfigured. Restart service; verify `/metrics` reachable on loopback. |
| **SOPS decrypt fails** (`no matching keys found`) | `ssh root@mcow 'test -f /root/.config/sops/age/keys.txt && echo OK'` | Missing age key on mcow. See `servers/mcow/README.md` for key material path; restore from operator hub key backup. |
| **Bun crashes on start** (Illegal instruction) | `ssh root@mcow 'bun --version'` and confirm `1.1.38`. | QEMU CPU lacks AVX ≥ 1.2.x; stay pinned on `1.1.38` per group_vars/all.yml. Do not bump without P-05 re-validation. |
| **rsync --delete wiped secrets/** | `ssh root@mcow 'ls /opt/homelab-admin/app/secrets/'` empty. | Regression of Plan 13-07 fix; `--exclude=secrets` must be present in deploy-homelab-admin.yml rsync_opts. Redeploy with full env stage. |
