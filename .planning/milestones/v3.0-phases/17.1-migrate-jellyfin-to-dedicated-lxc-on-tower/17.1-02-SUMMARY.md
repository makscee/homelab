## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 — Blocking] SSH host key for `jellyfin` not trusted on controller**
- Found during: running `verify-jellyfin-lxc.sh` first time — all `ssh root@jellyfin` checks failed with `Host key verification failed.`
- Fix: `ssh-keyscan -T 5 jellyfin >> ~/.ssh/known_hosts` (and 100.77.246.74).
- This is a one-time controller-local bootstrap; no repo artifact needed.

**2. [Rule 1 — Bug] `findmnt SIZE` regex mismatch**
- Found during: verify script run — `tmpfs size=4G` check failed because `findmnt -no SIZE` emits leading whitespace (`  4G`).
- Fix: relaxed anchor from `^4G$` to `(^|\s)4G\s*$`.
- File: `scripts/verify-jellyfin-lxc.sh`.

**3. [Rule 1 — Scope] vmbr1 health check was hard-fail but belongs to Plan 04**
- Found during: verify script run — `curl http://10.10.20.11:8096/health` times out from tower.
- Root cause: Jellyfin listens 0.0.0.0:8096 (confirmed via `ss -tlnp`). vmbr1 path is not configured end-to-end; that's Plan 04's `tower:22098 → 10.10.20.11:8096` reverse-proxy job.
- Fix: added a Tailnet-path check (`100.77.246.74:8096`) as the authoritative hard check, downgraded vmbr1 to `check_warn` with a pointer to Plan 04.

### Deferred (operator-approved)

**D-17 — Intel iGPU HW transcode**
- Root cause diagnosed: iGPU BIOS-disabled on tower (auto-off when dGPU present). NVIDIA RTX 2060 is sole DRM device. Full diagnosis + resume checklist in `17.1-02-IGPU-PROBE.md`.
- Operator decision 2026-04-18: defer BIOS toggle; test Jellyfin perf on CT 101 as-is (CPU-only transcode) before committing to wave-3 cutover.
- Resume path: physical BIOS access → enable `iGPU Multi-Monitor` → reboot → re-run `scripts/verify-jellyfin-lxc.sh`; when both warnings clear, promote the `check_warn` blocks at the bottom of the script to `check` (hard fail) to close D-17.
