# Phase 17.1 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.
> This is an ops/infra phase — validation is Ansible idempotence + SSH/shell probes + manual UAT, not a code test framework.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | ansible (check mode + PLAY RECAP) + shell probes (ssh, curl, pct, vainfo) |
| **Config file** | `ansible/inventory/homelab.yml` (to be extended with `jellyfin` host) |
| **Quick run command** | `ansible-playbook -i ansible/inventory/homelab.yml ansible/playbooks/deploy-jellyfin.yml --check` |
| **Full suite command** | `ansible-playbook -i ansible/inventory/homelab.yml ansible/playbooks/deploy-jellyfin.yml && bash scripts/verify-jellyfin-lxc.sh` |
| **Estimated runtime** | ~60–120s (quick), ~3–5min (full + probes) |

---

## Sampling Rate

- **After every task commit:** Ansible `--check` on affected play; `pct status 101` if LXC touched
- **After every plan wave:** Full `deploy-jellyfin.yml` + probe script (Tailscale reachable, systemd active, /health 200, vainfo lists iGPU, mp0/mp1 readable)
- **Before `/gsd-verify-work`:** Manual UAT = user plays 1080p + 4K to real client, no buffering
- **Max feedback latency:** ~120s for automated probes; UAT latency is user-dependent

## Idempotence Gate

- [ ] Running `deploy-jellyfin.yml` twice in a row produces `PLAY RECAP … changed=0 failed=0` on the second run (except for documented exceptions like service-restart handlers when config genuinely changed).
- [ ] Running the cutover rsync task twice does not double-copy or re-chown files unnecessarily.

---

## Rollback Validation

- [ ] During soak week: stopping `systemctl stop jellyfin` on LXC 101 + starting the old Docker container on LXC 100 + flipping tower:22098 proxy → `docker-tower:8096` restores service in <5 min. Rollback drill must be rehearsed at least once during the soak window.
