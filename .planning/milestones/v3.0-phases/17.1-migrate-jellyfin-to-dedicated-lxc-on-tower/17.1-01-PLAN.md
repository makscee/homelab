<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| tower host ↔ LXC 101 | kernel boundary — unprivileged mapping prevents LXC root = host root |
| LXC 101 ↔ /mnt/pve/{wdc,sea} | RO bindmount — LXC cannot modify media (prevents ransomware-style corruption from compromised Jellyfin) |
| LXC 101 ↔ /dev/dri/renderD128 | device passthrough — render node only (no card0 display access) |

## STRIDE Threat Register

| ID | Category | Component | Disposition | Mitigation |
|----|----------|-----------|-------------|------------|
| T-17.1-01 | Elevation of Privilege | LXC 101 root breakout to tower host | mitigate | `unprivileged: 1` + `features: nesting=1,keyctl=1` (no `privileged=1`, no `/dev/kmsg`, no custom idmap escalations). ASVS L1 §V14 server hardening. |
| T-17.1-02 | Tampering | Compromised Jellyfin writes to /media | mitigate | mp0/mp1 with `ro=1` — kernel-enforced read-only even if LXC root is compromised. |
| T-17.1-03 | Information Disclosure | /dev/dri/card0 display-output access | mitigate | Pass only `renderD128` via `dev0:`; card0 NOT in config. |
| T-17.1-04 | Spoofing | Non-Tailnet attacker hits 10.10.20.11:8096 | accept | vmbr1 is internal L2 only; tower ingress (Plan 04) gates external. |
| T-17.1-05 | Denial of Service | LXC 101 exhausts tower RAM | mitigate | `memory: 8192` hard cap + `swap: 512`. |
</threat_model>
