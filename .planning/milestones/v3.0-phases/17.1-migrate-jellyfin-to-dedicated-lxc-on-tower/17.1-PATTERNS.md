## Pattern Assignments

### `servers/tower/lxc-101-jellyfin.conf` (config, declarative-state)

**Primary analog:** `servers/tower/lxc-204-cc-worker.conf` (unprivileged skeleton)
**Secondary analog:** `servers/tower/lxc-100-docker-tower.conf` (iGPU passthrough + mp0/mp1 — legacy form only; prefer `dev0:` per RESEARCH)

**Skeleton pattern (copy from `lxc-204-cc-worker.conf`):**
```
arch: amd64
cores: 4
features: nesting=1
hostname: cc-worker
memory: 4096
nameserver: 1.1.1.1
net0: name=eth0,bridge=vmbr1,gw=10.10.20.1,hwaddr=BC:24:11:97:30:D3,ip=10.10.20.204/24,type=veth
ostype: debian
rootfs: local-lvm:vm-204-disk-0,size=16G
swap: 512
timezone: Europe/Moscow
unprivileged: 1
```
**Adapt for CT 101:** `cores: 4` (keep), `memory: 8192`, `rootfs: toshi:...,size=32G`, `ip=10.10.20.11/24`, `hostname: jellyfin`, `features: nesting=1,keyctl=1`, drop `net1` (D-03 — no vmbr0). Add `onboot: 1` + `tags: jellyfin,media,homelab`.

**Bindmount + iGPU pattern (copy from `lxc-100-docker-tower.conf` lines 7-8, 29-31):**
```
mp0: /mnt/pve/wdc,mp=/media/wdc
mp1: /mnt/pve/sea,mp=/media/sea
...
lxc.cgroup2.devices.allow: c 226:0 rwm
lxc.cgroup2.devices.allow: c 226:128 rwm
lxc.mount.entry: /dev/dri dev/dri none bind,optional,create=dir
```
**Adapt for CT 101:** add `,ro=1` to mp0/mp1 (D-07). Replace the 3 legacy iGPU lines with the modern Proxmox 8.2 one-liner per RESEARCH §Key Findings #2:
```
dev0: /dev/dri/renderD128,gid=<HOST_RENDER_GID>
```
Fallback to the legacy 3-line form only if `dev0:` misbehaves.

**Anti-pattern:** Do NOT hand-edit the repo file — always capture via `ssh root@tower 'pct config 101' > servers/tower/lxc-101-jellyfin.conf` after Proxmox-side changes (RESEARCH §Anti-Patterns).

### `scripts/verify-jellyfin-lxc.sh` (utility, request-response)

**Intel iGPU detection pattern** (lines 7-14) — copy verbatim, useful for confirming `renderD128` is Intel on tower:
```bash
INTEL_RENDER=$(ssh root@tower 'for d in /sys/class/drm/renderD*; do
  vendor=$(cat "$d/device/vendor" 2>/dev/null)
  if [ "$vendor" = "0x8086" ]; then
    basename "$d"
    exit 0
  fi
done; echo "NOT_FOUND"')
```

**LXC-side DRI check pattern** (lines 26-33) — adapt `ssh root@docker-tower` → `ssh root@jellyfin`:


**vainfo pattern** (line 37) — adapt: drop `docker exec jellyfin`, use native binary path:
```bash
ssh root@jellyfin "/usr/lib/jellyfin-ffmpeg/vainfo --display drm --device /dev/dri/$INTEL_RENDER 2>&1 | tail -20"
```
