## Spacing Scale

Declared values (multiples of 4 — Tailwind defaults, no custom extension):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`p-1`, `gap-1`) | Kebab icon gap, badge inner padding, sparkline dot padding |
| sm | 8px (`p-2`, `gap-2`) | Progress bar wrapper padding, inline form helper-text gap |
| md | 16px (`p-4`, `gap-4`) | Table cell padding, form field vertical rhythm, dialog body padding |
| lg | 24px (`p-6`, `gap-6`) | Page section padding, dialog outer padding, card padding |
| xl | 32px (`p-8`) | Page content container padding, detail-page chart wrapper |
| 2xl | 48px (`p-12`) | Empty-state vertical padding when no tokens exist |
| 3xl | 64px (`p-16`) | Not used this phase |

**Exceptions:**
- Table row min-height fixed at 56px (multiple of 4) to accommodate inline 5h bar + 7d bar + sparkline without clipping on dense data.
- Sparkline column fixed-width 96px (multiple of 4) to keep row geometry stable across varying data.
- Kebab trigger is a 32×32 hit target (`h-8 w-8`) to meet touch-size while staying compact inside the row.

## Typography

Slate preset inherits Tailwind defaults. Declared roles (3 sizes, 2 weights):

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 (`leading-6`) |
| Label / table header | 14px (`text-sm`) | 600 (semibold) | 1.4 |
| Heading (page h1, dialog title) | 20px (`text-xl`) | 600 (semibold) | 1.3 |
| Display (detail-page token label) | 28px (`text-3xl`) | 600 (semibold) | 1.2 |

**Monospace exception:** token ID, `sk-ant-oat01-...` format hint, and audit `diff` previews use `font-mono text-xs` (12px) — necessary for alignment and to signal "raw data, do not edit". This is an intentional fourth size reserved for code-like content only.

## Color

Inherits Phase 12 dark-default CSS variables (see `apps/admin/app/globals.css`). 60/30/10 split:

| Role | Value (HSL var) | Usage |
|------|-----------------|-------|
| Dominant (60%) | `--background` `222.2 84% 4.9%` | Page background, table body |
| Secondary (30%) | `--secondary` / `--muted` `217.2 32.6% 17.5%` | Table header row, card surfaces, sidebar (carried), dialog backdrop, skeleton loaders |
| Accent (10%) | `--primary` `210 40% 98%` (near-white on dark) | Primary CTA button fill, active tab, sparkline stroke, progress-bar fill under threshold |
| Destructive | `--destructive` `0 62.8% 30.6%` | Delete button, delete confirmation, rotate confirmation button, "≥95% threshold" bar state |

**Accent reserved for:**
1. The single primary CTA per view — `Add token` button on the `/tokens` index, `Save` button in each dialog.
2. Sparkline line stroke in the table row (visual anchor for at-a-glance scan).
3. Progress-bar fill when value is `< 0.80` (safe zone).
4. Focus ring on form inputs (via `--ring`).

**Threshold color map for utilization bars** (driven by D-13-02 dashboard-feel requirement):
- `< 0.80` → accent / primary (safe)
- `>= 0.80 < 0.95` → warning (`amber-500` `38 92% 50%` — declared here since base palette lacks a warning token)
- `>= 0.95` → destructive

The amber warning tone is the only color added beyond the existing palette and is used **only** on the two progress bars (5h, 7d). It is not a general-purpose token.

**Enabled/disabled badge:**
- Enabled → `--secondary` background, `--foreground` text (neutral, default)
- Disabled → `--muted` background, `--muted-foreground` text (dimmed)

No color drift beyond what is listed here.

## Interaction Contract

### Table (list view)

- **Sortable columns** (D-13-01): Label, Tier, Owner, 5h %, 7d %. Sort toggles via header click; direction indicator uses `lucide-react` `ArrowUp` / `ArrowDown` next to header.
- **Row hover** applies `--secondary` background to signal target.
- **Label click** navigates to `/tokens/[id]` detail (D-13-04). Entire label cell is the link target (not wrapping the full row, to leave the kebab and sort focus rings clean).
- **Kebab (`⋯`)** uses `lucide-react` `MoreVertical`; opens `DropdownMenu` with items in this order: `Rotate`, `Disable` / `Enable` (label flips based on state), `Rename`, `Delete`. `Delete` is styled with destructive text color.
- **Keyboard:** full row operable via keyboard — tab order is Label-link, then Kebab trigger; arrow keys navigate the dropdown.
- **Focus ring** uses `--ring` on every interactive element (no custom ring).

### Add-token dialog

- Triggered by page-level `Add token` CTA; opens shadcn `Dialog`.
- Token-value input `type="password"` with an eye-toggle (`Eye` / `EyeOff` lucide icons) to reveal/hide. Reveal is client-state only; value is never echoed in server responses.
- Submit disables button and shows inline spinner until server action resolves.
- On success: dialog closes, row appears on next `router.refresh()`, success toast fires.
- On error: dialog stays open, inline field error shows, error toast fires. Token value is retained (so operator doesn't re-paste on a typo in label/host).

### Rotate dialog

- Triggered from kebab → `Rotate`.
- Confirmation pattern is a **single-field dialog** (not a full re-open of the add form). Label, Owner, Tier are carried over automatically.
- Confirm button destructive-colored to reflect the immediate invalidation of the prior value.

### Delete dialog

- `AlertDialog` with typed-label confirmation (D-13-12). Confirm stays disabled until text input matches `label` exactly (case-sensitive).
- Soft-delete by default (D-13-12); UI copy says "deleted" — soft-vs-hard is invisible to the operator.

### Toasts

- Position: top-right (shadcn default).
- Duration: 4s for success, 6s for error (errors linger long enough to read the next-step text).
- Max stack: 3 visible.

### Degraded mode (D-13-10)

- `Alert` component pinned above the table.
- Kebab triggers and `Add token` CTA are rendered with `aria-disabled="true"` and `cursor-not-allowed` styling; click is a no-op. Tooltip on hover explains `SOPS write unavailable`.
- Live gauges continue to render (read path works via exporter snapshot).

### Detail page (`/tokens/[id]`)

- Breadcrumb: `Tokens / {label}` (lucide `ChevronRight` separator).
- Header row: display-size label, tier badge, owner host, enabled/disabled badge, kebab with the same action set as list view.
- Primary content: Recharts line chart, 7-day window, single series, y-axis 0-100%, x-axis time with day labels.
- Chart colors: stroke `--primary`, fill `--primary` at 10% opacity, threshold reference lines at 80% (amber) and 95% (destructive), dotted.
- Chart empty state (only when exporter genuinely has no range data): inline `Skeleton` with caption `No range data yet. Check exporter health.`
- Secondary row: metadata card — `id`, `added_at`, `rotated_at` (if set), `notes` (if set). `id` and timestamps rendered in monospace.

## Accessibility

- Dialogs and AlertDialogs inherit Radix focus-trap + `Esc` dismiss + `aria-labelledby` / `aria-describedby` (shadcn defaults; do not override).
- Progress bars expose `role="progressbar"` with `aria-valuenow`, `aria-valuemin=0`, `aria-valuemax=100`, and an `aria-label` ending in the raw percentage (`"5h usage: 62%"`) so screen readers don't rely on color-threshold distinction.
- Color is never the sole signal for the 80% / 95% thresholds — each bar carries its numeric percentage to the right of the bar track.
- Destructive actions always require a second step (confirmation dialog or typed confirmation); no destructive one-click in the kebab.
- Token-value inputs never auto-focus copy-to-clipboard (operator must explicitly request reveal).
