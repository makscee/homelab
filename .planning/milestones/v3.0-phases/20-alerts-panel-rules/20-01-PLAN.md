## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /api/alerts/list | untrusted session cookie crosses here; Auth.js session validation enforces identity |
| mcow Next.js → Prometheus (Tailnet HTTP) | Prometheus on docker-tower is Tailnet-only; URL is server-env only |
| Prometheus labels → React render | label VALUES are untrusted (come from exporters) — XSS vector if interpolated into HTML |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-01-01 | S (Spoofing) | /api/alerts/list | mitigate | `const session = await auth(); if (!session) return 401` — clone of count/route.ts pattern |
| T-20-01-02 | T (Tampering) | severity label → SeverityBadge className | mitigate | Enum-bucket severity to {critical,warning,info,other} in alerts-list.server.ts; never pass raw label into className |
| T-20-01-03 | T (Tampering) | arbitrary label values rendered in LabelsCell | mitigate | React JSX auto-escaping on `{k}={v}` text; no `dangerouslySetInnerHTML`; values pass through as text only |
| T-20-01-04 | I (Info disclosure) | PROMETHEUS_URL / ALERTMANAGER_URL leaking to client bundle | mitigate | `import "server-only"` sentinel on `*.server.ts`; env read in RSC page only, never exposed as `NEXT_PUBLIC_*` (Pitfall 5) |

| T-20-01-06 | E (Elevation) | unauthed user reaches /alerts page | mitigate | `auth()` gate in RSC with `redirect("/login")` on miss (audit/page.tsx pattern) |
