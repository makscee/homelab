# Milestone v1.0: Homelab Infrastructure-as-Code

**Status:** SHIPPED 2026-04-15
**Phases:** 01-04 (4 phases)
**Total Plans:** 18

## Overview

The v1.0 milestone established the homelab as an infrastructure-as-code repository — the single source of truth for 4 Tailnet hosts plus Proxmox LXCs (6 monitored hosts total). Starting from a patchwork of configs and tribal knowledge, this milestone delivered: SOPS+age secrets, full server inventories, reproducible service definitions (Docker Compose + LXC configs + AmneziaWG), a Prometheus/Grafana/Alertmanager/node-exporter monitoring stack on mcow (operator plane) + docker-tower (data plane), an operator overview dashboard pinned as Grafana home, and proven end-to-end Telegram alert delivery.

**Core value achieved:** any server's full stack can be reliably reproduced from this repo alone.

## Milestone Summary

**Key Accomplishments:**

1. **Secrets foundation (Phase 1)** — SOPS+age pattern live at hub level; .gitignore blocks plaintext secrets; canonical `secrets/`, `servers/`, `docs/` structure committed.
2. **Every service reproducible (Phase 2)** — Docker Compose with pinned tags on docker-tower; mcow systemd units audited; LXC configs captured for all 6 VMIDs; AmneziaWG template with SOPS keys; `scripts/tailscale-provision.sh` seeded.
3. **Monitoring pipeline (Phase 3)** — node-exporter on all active hosts via Ansible playbook; Prometheus + Alertmanager + cAdvisor in compose; Grafana datasource + dashboards provisioned; `scripts/healthcheck.sh` + promtool rule tests + phase gate harness.
4. **Operator plane on mcow (Phase 4)** — Grafana 12.3.1 + Alertmanager migrated off docker-tower to mcow (Tailscale-only); operator overview dashboard pinned as home showing all 6 hosts at a glance; docker-tower volumes preserved 7d for rollback.
5. **Alert pipeline proven E2E (Phase 4)** — real Alertmanager rule fire → Telegram delivery confirmed via `@void_homelab_bot`; `alertmanager_notifications_total{integration="telegram"}` went 0→1→2 with zero `_failed_total`; Tailscale App Connector on nether deployed as IPv4 Telegram egress fallback.
6. **6/6 host coverage** — animaya-dev SSH unblocked via `pct exec` from tower; node-exporter deployed; last monitoring gap closed.

**Key Decisions:**

- Claude Code as operator (runbooks use numbered steps, exact hostnames, expected outputs, stop conditions).
- SOPS + age for secrets at hub level; homelab repo references only encrypted values, never raw.
- Document-first, automate-second — capture what exists before scripting.
- AmneziaVPN only on nether — XRay/VLESS out of scope.
- Operator plane on mcow (Grafana + Alertmanager); data plane on docker-tower (Prometheus + cAdvisor + media services).
- Per-server directory layout: `servers/{hostname}/` with `inventory.md`.
- SOPS naming convention: `*.sops.yaml` = encrypted (committed), `*.yaml` = plaintext (blocked).
- Tailscale App Connector on nether for IPv4 Telegram egress (Moscow IPv6-only path blocked by Telegram).
