## Spacing Scale

Declared values (must be multiples of 4). Using Tailwind v4 CSS-first spacing utilities mapped to 8-point scale:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding, table cell icon gap |
| sm | 8px | Table cell padding (compact dense table), badge margin |
| md | 16px | Page content padding, column header padding |
| lg | 24px | Page section gap, card padding |
| xl | 32px | Page top margin, table container margin |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (unused in dense ops table) |

Exceptions: Touch targets on severity badge column — minimum 44px row height for tap accessibility even in dense mode.

## Color

Dark-only theme. HSL values from shadcn slate CSS variables (ui-style-spec.md §2, locked).

| Role | CSS Token | Usage |
|------|-----------|-------|
| Dominant (60%) | `--background` `hsl(222.2 84% 4.9%)` | Page background, table row background |
| Secondary (30%) | `--card` / `--secondary` `hsl(217.2 32.6% 17.5%)` | Table header row, stale/error banner background, column header bg |
| Accent (10%) | `--primary` `hsl(210 40% 98%)` | Sort indicator icon (active column only) |
| Destructive | `--destructive` `hsl(0 62.8% 30.6%)` | Not used in this read-only phase (no mutation actions) |

Accent reserved for: active sort indicator icon only. No other elements use accent.

### Severity Badge Palette

Severity badges use inline color overrides on top of the shadcn `<Badge>` primitive. These are the only semantic colors beyond the base palette:

| Severity | Background | Text | Tailwind classes |
|----------|------------|------|-----------------|
| critical | red-900/30 + red-500 border | red-400 | `bg-red-900/30 border-red-500/50 text-red-400` |
| warning | amber-900/30 + amber-500 border | amber-400 | `bg-amber-900/30 border-amber-500/50 text-amber-400` |
| info | slate-700/50 + slate-500 border | slate-300 | `bg-slate-700/50 border-slate-500/50 text-slate-300` |
| (unknown) | muted bg | muted-foreground | `bg-muted text-muted-foreground` |

Source: CONTEXT.md `<specifics>` — "critical = red, warning = amber, info = slate (matches ui-style-spec tokens)".

## Interaction Contract

### Auto-refresh
- SWR `refreshInterval: 15_000` (15s). Source: CONTEXT.md D-03 (discretion exercised — 15s fresh enough, low load).
- On error: `healthy: false` returned by API, SWR does NOT retry-storm. Banner appears with `stale_since` timestamp.
- Stale indicator: `<StaleDot>` component pattern already exists in `app/(auth)/_components/StaleDot.tsx` — reuse or reference.

### Sorting
- Client-side sort on Severity, Alert Name, Duration columns.
- Sort state: React `useState` — no URL persistence (operator preference, not bookmark-worthy).
- Default: Duration descending.

### Labels expand/collapse
- Per-row toggle via React `useState` on row key.
- Expanded: renders `key=value` pairs as `<span>` chips, `text-xs bg-secondary rounded px-1`.
- No animation required.

### Nav Alert Badge
- `<NavAlertBadge>` already exists at `app/(auth)/_components/NavAlertBadge.tsx`.
- Consumes `/api/alerts/count` — cadence unchanged (whatever Phase 14-05 established; planner confirms).
- No change to badge in this phase unless planner finds cadence trivially alignable to 15s.

### Loading State
- 5 skeleton rows (`<Skeleton>`) at table row height while initial SWR fetch in flight.
- Skeleton columns match real column widths.

## API Contract (for executor reference)

`GET /api/alerts/list` returns:

```ts
type AlertsListResponse = {
  healthy: boolean;
  stale_since?: string; // ISO8601, present only when healthy: false
  alerts: {
    alertname: string;
    severity: "critical" | "warning" | "info" | string;
    instance: string;
    summary: string;
    duration_seconds: number;
    labels: Record<string, string>;
  }[];
};
```

Route follows `apps/admin/app/api/alerts/count/route.ts` pattern exactly: `runtime: "nodejs"`, `dynamic: "force-dynamic"`, session-gated via `auth()`, always HTTP 200.
