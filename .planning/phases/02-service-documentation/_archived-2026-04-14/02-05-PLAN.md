    ## Recreate a Container

    Proxmox doesn't accept a raw `pct config` file as input to `pct create` directly. The supported flow is:

    1. Create with the original template used, matching OSTYPE and ARCH from the conf file:
       ```
       ssh root@tower 'pveam list local'   # find template
       ssh root@tower 'pct create <VMID> local:vztmpl/<template>.tar.zst \
         --hostname <name> \
         --cores <N> \
         --memory <MB> \
         --net0 <net_line_from_conf> \
         --rootfs <storage>:<size> \
         --unprivileged <0|1> \
         --features <features_line>'
       ```
    2. Apply additional lines via `pct set <VMID> --<key> <value>` for each line in the .conf file NOT covered by create flags.
    3. Verify: `pct config <VMID>` on tower should produce output matching the repo .conf file.

      lxc.mount.entry: /dev/dri dev/dri none bind,optional,create=dir
