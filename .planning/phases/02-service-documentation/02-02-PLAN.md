    Step 1 — Extract authoritative tags from LIVE-AUDIT.md:
    Grep the docker-tower `### docker inspect (name|config-image|digest)` section for each service in the 3 existing compose files. The format is `/container_name|image:tag|sha256:digest`.

    Step 2 — For each service in each of the 3 existing compose files, replace the `image:` line with the exact tag from LIVE-AUDIT.md:
    - If LIVE-AUDIT shows `jellyfin/jellyfin:10.9.6` → write `image: jellyfin/jellyfin:10.9.6`
    - If LIVE-AUDIT shows `lscr.io/linuxserver/radarr:latest` with digest `sha256:abc123...` → write `image: lscr.io/linuxserver/radarr@sha256:abc123...` (digest pin, per Pattern 2)
    - NEVER leave `:latest` in a committed file

    Do this for EVERY service block in:
    - `servers/docker-tower/docker-compose.services.yml`
    - `servers/docker-tower/docker-compose.homestack.yml`
    - `servers/docker-tower/docker-compose.monitoring.yml`

    Step 3 — Remove `version:` top-level keys if present (State of the Art — deprecated in Compose Spec v2+).

    Step 4 — Secrets migration for monitoring stack (Pitfall 5):
    If `docker-compose.monitoring.yml` contains `GF_SECURITY_ADMIN_PASSWORD=REDACTED` (or any `*_PASSWORD=`, `*_TOKEN=`, `*_SECRET=` line with a non-placeholder value):
    a. Replace the inline env block for grafana with:
       ```yaml
       env_file:
         - /etc/homelab/docker-tower-monitoring.env
       ```
       (Operator decrypts the sops file to that path at deploy time — document in README.)
    b. Create `secrets/docker-tower-monitoring.sops.yaml` at hub level (one directory up from repo root — path: `../secrets/docker-tower-monitoring.sops.yaml` if cwd is the homelab repo; use the hub-level secrets directory). Contents before encryption:
       ```yaml
       GF_SECURITY_ADMIN_USER: admin
       GF_SECURITY_ADMIN_PASSWORD: <real password extracted from LIVE-AUDIT or ssh into docker-tower 'docker inspect grafana' env>
       ```
    c. Encrypt in place: `sops --encrypt --in-place secrets/docker-tower-monitoring.sops.yaml` (resolve the correct hub-level path; if hub secrets live at `$HOME/hub/secrets/` or `../../secrets/` relative to homelab repo root, use that).
    d. Verify: `grep -q 'sops:' secrets/docker-tower-monitoring.sops.yaml` succeeds.

    Do NOT commit plaintext secrets at any point. If Step 4 fails for any reason, stop and surface the failure — do not commit a compose file that still references a plaintext password.
  </action>

| compose env blocks → container runtime | Values leak to `docker inspect` output — still bad if committed but at least scoped to operator |
