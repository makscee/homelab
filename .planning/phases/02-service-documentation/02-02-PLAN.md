    PART B — Grafana password to SOPS (Pitfall 5, SEC-01):
    1. Determine the running Grafana admin password:
       ```bash
       ssh root@docker-tower "docker inspect grafana --format '{{range .Config.Env}}{{println .}}{{end}}'" \
         | grep '^GF_SECURITY_ADMIN_PASSWORD='
       ```
       Capture the value (not into a file in repo; keep only in shell vars).
    2. Identify age recipient: if `.sops.yaml` exists at repo root, read the `age:` key. Otherwise use the recipient from the existing `secrets/test.sops.yaml` header.
    3. Create an unencrypted temp file `/tmp/phase02-ev/docker-tower.env`:
       ```
       GF_SECURITY_ADMIN_USER=admin
       GF_SECURITY_ADMIN_PASSWORD=<value from ssh>
       ```
    4. Encrypt with SOPS to `secrets/docker-tower.sops.yaml`:
       ```bash
       sops --encrypt --age <recipient> --input-type dotenv --output-type yaml \
         /tmp/phase02-ev/docker-tower.env > secrets/docker-tower.sops.yaml
       shred -u /tmp/phase02-ev/docker-tower.env 2>/dev/null || rm -f /tmp/phase02-ev/docker-tower.env
       ```
       (Use `--input-type dotenv --output-type yaml` so operators decrypt to yaml.)
       If the operator toolchain requires plain yaml input instead, construct a yaml file with the two keys and `sops --encrypt --age <recipient> --in-place secrets/docker-tower.sops.yaml`.
    5. Edit `docker-compose.monitoring.yml` Grafana service: remove the `GF_SECURITY_ADMIN_PASSWORD` line from `environment:`; the operator workflow is to `sops --decrypt secrets/docker-tower.sops.yaml > /run/secrets/grafana.env` then `env_file: [/run/secrets/grafana.env]` on the service. Add the service's `env_file:` block accordingly AND add a top-of-file comment block:
       ```yaml
       # Secrets: before `docker compose up`, run:
       #   sops --decrypt ../../secrets/docker-tower.sops.yaml > /run/secrets/grafana.env
       # File is NOT committed (handled by .gitignore /run/secrets/).
       ```
    6. Ensure `.gitignore` (at repo root) excludes `/run/secrets/` — if already covered by earlier Phase 1 rules, no change; if missing, add the pattern.
