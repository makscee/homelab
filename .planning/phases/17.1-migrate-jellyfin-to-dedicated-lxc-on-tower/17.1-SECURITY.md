## Notes

**T-18 ingress shape deviation:** Plan 04 assumed nginx on :22098; implementation used iptables DNAT on vmbr0:8096 + socat :22098 systemd unit. T-17.1-18 (config drift) is CLOSED because: (a) the canonical rule is committed to `servers/tower/ingress/jellyfin.iptables`, (b) the socat unit is documented in `servers/tower/ingress/README.md`, and (c) pre-change iptables backup exists at `/root/pre-17.1-iptables.save` on tower. The deviation in ingress mechanism does not affect the threat's mitigation coverage — the goal was IaC commitment of the ingress config, which is met.

**T-16 (Host injection) note:** T-17.1-16 assumed nginx `proxy_set_header Host $host` as the mitigation. The actual implementation uses L4 DNAT (no L7 proxy), which means Host headers pass through unchanged but there is also no wildcard server block that could be spoofed. The threat is CLOSED under the actual architecture — no Host-based routing exists to bypass.

**D-17 / iGPU deferral:** Not a security threat. The HW transcode deferral (BIOS iGPU disabled) affects only performance, not any STRIDE threat. The two WARN blocks in `scripts/verify-jellyfin-lxc.sh` (lines 92–100) are ready to promote to hard checks when the BIOS window occurs.
