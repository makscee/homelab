<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| docker-tower ↔ jellyfin (LXC 100 ↔ LXC 101) | rsync over SSH over Tailnet — encrypted in transit |
| SQLite DBs during rsync | at-rest plaintext includes user account records (hashes) |

## STRIDE Threat Register

| ID | Category | Component | Disposition | Mitigation |
|----|----------|-----------|-------------|------------|
| T-17.1-11 | Tampering | SQLite WAL/SHM copied mid-write → corruption | mitigate | `docker stop` (NOT pause) before rsync; verified via `docker inspect State.Status == exited`. |
| T-17.1-12 | Information Disclosure | rsync path exposes password hashes to network | mitigate | rsync over SSH over Tailscale WireGuard — double-encrypted; no plaintext egress. |
| T-17.1-13 | Repudiation | No audit of what was copied | mitigate | CUTOVER-LOG.md records timestamps + row-count deltas. |
| T-17.1-14 | Denial of Service | Service unavailable during cutover | accept | Gated by Task 1 maintenance window; hot-standby rollback available for 1 week (D-14/D-16). |
| T-17.1-15 | Elevation of Privilege | Post-rsync file ownership mismatch allows non-jellyfin user to read DB | mitigate | `chown -R jellyfin:jellyfin` + default umask 0755/0644. |
</threat_model>
