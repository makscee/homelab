<task type="auto">
  <name>Task 1: Author provision-proxmox-dashboard-token.yml (idempotent role/user/token + CA capture + SOPS write)</name>
  <files>ansible/playbooks/provision-proxmox-dashboard-token.yml</files>
  <action>
    Create new playbook targeting `hosts: tower` with `become: true`. Implement per RESEARCH §1 + §2 verbatim with these exact tasks (in order):

    1. `pveum role add DashboardReadOnly -privs "VM.Audit,Datastore.Audit"` — register, failed_when rc != 0 AND 'already exists' not in stderr; changed_when rc == 0. (Per D-03 — strictly read-only; no VM.PowerMgmt.)
    2. `pveum user add dashboard-operator@pve --comment 'mcow homelab-admin read-only'` — same idempotence pattern.
    3. `pveum acl modify / -user dashboard-operator@pve -role DashboardReadOnly` — changed_when: false (pveum acl modify is a SET).
    4. `pveum user token list dashboard-operator@pve --output-format json` — register token_list, changed_when: false. (Idempotence guard per CONTEXT requirement: re-runs do not rotate secret.)
    5. `pveum user token add dashboard-operator@pve readonly --privsep 0 --output-format json` — register token_create, `when: 'readonly' not in (token_list.stdout | from_json | map(attribute='tokenid') | list)`, no_log: true.
    6. `set_fact: proxmox_token_secret: "{{ (token_create.stdout | from_json).value }}"` — `when: token_create is changed`, no_log: true.
    7. `ansible.builtin.slurp: src: /etc/pve/pve-root-ca.pem` → register pve_ca.
    8. `delegate_to: localhost`, `become: false` — write `proxmox_dashboard.tower_ca_pem` into `secrets/mcow.sops.yaml` using `sops set` with the CA content (b64decoded, JSON-quoted via `jq -Rs .`). Always run (CA never rotates per session — but `sops set` is itself idempotent on identical input).
    9. `delegate_to: localhost`, `become: false` — write `proxmox_dashboard.token_secret` into `secrets/mcow.sops.yaml`, `when: proxmox_token_secret is defined`, no_log: true.

    Header doc-comment must include: `# Idempotent: re-run yields all-ok and DOES NOT rotate token. To rotate, manually pveum user token remove first.`

    Use `pveum` shell rather than `community.proxmox` — per RESEARCH §1 + Assumption A1, no `proxmox_user_token` module ships in the collection.
  </action>
  <verify>
    <automated>cd ansible && ansible-playbook -i inventory/homelab.yml playbooks/provision-proxmox-dashboard-token.yml --check --diff</automated>
  </verify>
