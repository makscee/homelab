```yaml
---
# Decrypt homelab_admin secrets from secrets/mcow.sops.yaml on the Ansible controller
# (localhost), then render /etc/homelab-admin/env on the target host with mode 0600
# owned by homelab-admin:homelab-admin.
#
# Consumers: ansible/playbooks/deploy-homelab-admin.yml (Plan 12-09) includes this via
#   - include_tasks: tasks/homelab-admin-secrets.yml
#
# Preconditions satisfied by caller playbook:
#   - group_vars/all.yml provides: homelab_admin_env_dir, homelab_admin_service_user,
#     homelab_admin_service_group
#   - Target host already has the homelab-admin system user + group created
#   - Controller has sops + age installed, age key at ~/.config/sops/age/keys.txt
#   - Caller provides homelab_admin_commit_sha via --extra-vars or set_fact before include
#
# Security: no_log: true on decrypt; no raw secret values in Ansible stdout/log/facts.

- name: Decrypt homelab_admin secrets locally
  delegate_to: localhost
  ansible.builtin.shell:
    cmd: "sops --decrypt --extract '[\"homelab_admin\"]' --output-type json {{ playbook_dir }}/../../secrets/mcow.sops.yaml"
  register: _hla_sops_out
  changed_when: false
  check_mode: false
  no_log: true

- name: Parse decrypted homelab_admin secrets
  ansible.builtin.set_fact:
    _hla_secrets: "{{ _hla_sops_out.stdout | from_json }}"
  no_log: true

- name: Ensure /etc/homelab-admin env directory exists on target
  ansible.builtin.file:
    path: "{{ homelab_admin_env_dir }}"
    state: directory
    owner: root
    group: "{{ homelab_admin_service_group }}"
    mode: '0750'

- name: Render /etc/homelab-admin/env on target
  ansible.builtin.copy:
    content: |
      # Rendered by Ansible from secrets/mcow.sops.yaml (Plan 12-08). Do NOT hand-edit.
      AUTH_SECRET={{ _hla_secrets.auth_secret }}
      AUTH_URL=https://homelab.makscee.ru
      HOMELAB_ADMIN_GITHUB_OAUTH_CLIENT_ID={{ _hla_secrets.github_oauth_client_id }}
      HOMELAB_ADMIN_GITHUB_OAUTH_CLIENT_SECRET={{ _hla_secrets.github_oauth_client_secret }}
      HOMELAB_ADMIN_ALLOWED_GITHUB_LOGINS={{ _hla_secrets.allowed_github_logins }}
      HOMELAB_ADMIN_COMMIT_SHA={{ homelab_admin_commit_sha | default('unknown') }}
    dest: "{{ homelab_admin_env_dir }}/env"
    owner: "{{ homelab_admin_service_user }}"
    group: "{{ homelab_admin_service_group }}"
    mode: '0600'
  no_log: true
  notify: Restart homelab-admin

- name: Drop in-memory secret facts after writing env file
  ansible.builtin.set_fact:
    _hla_secrets: null
    _hla_sops_out: null
  no_log: true
```

<threat_model>
## Trust Boundaries
| Boundary | Description |
|----------|-------------|
| git repo → operator laptop | SOPS ciphertext crosses this boundary; decrypts only with age key |
| Ansible controller → mcow | Plaintext env value transits Ansible's SSH connection to be written on target |
| /etc/homelab-admin/env on mcow → homelab-admin process | Service reads at startup |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-08-01 | Information Disclosure | SOPS ciphertext compromise → offline brute force | accept | age encryption is post-quantum-soft but operator-managed key is a single point of failure. Residual. Rotation path documented in `docs/setup-github-oauth.md`. |
| T-12-08-02 | Information Disclosure | Secret values echo into Ansible stdout/log/facts (v1.0 AM-04-style leak) | mitigate | `no_log: true` on every task that touches the decrypted value (decrypt, set_fact, render); final set_fact drops from memory |
| T-12-08-03 | Information Disclosure | Secret values leak into git diff | mitigate | Task 1 is a human-only step in `sops` editor (SOPS handles encryption on save); Claude Code never sees plaintext. Acceptance criterion greps git history for plaintext markers. |
| T-12-08-04 | Information Disclosure | `/etc/homelab-admin/env` world-readable | mitigate | `mode: '0600'`, `owner: homelab-admin`, `group: homelab-admin`. Parent dir `mode: '0750'` with group ownership to homelab-admin. No other user on mcow can open. |
| T-12-08-05 | Tampering | `/etc/homelab-admin/env` modified out-of-band → app picks up malicious values | mitigate | Ansible replaces the file on every deploy (`copy` module is idempotent — unchanged content → no change; tampered content → overwritten + `notify: Restart`). Phase 17 adds file-integrity check. |
| T-12-08-06 | Repudiation | SOPS edit has no author metadata | accept | Git commit records who added the encrypted block (though not who last decrypted). Acceptable for 1-operator threat model. |

Residual: LOW. Biggest residual is age private key theft — addressed outside this plan (OS-level secret hygiene on operator laptop).
</threat_model>
