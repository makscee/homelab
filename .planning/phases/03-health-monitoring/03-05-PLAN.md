    **Step 1 — Create `scripts/healthcheck.sh` as a production-grade bash CLI.**

    Header comment block:
    ```
    # scripts/healthcheck.sh — Query Prometheus HTTP API to verify host health.
    # Usage:
    #   scripts/healthcheck.sh <hostname>         # single host
    #   scripts/healthcheck.sh --all              # all 6 hosts
    #   scripts/healthcheck.sh --help
    # Exit: 0=ok, 1=degraded (warnings), 2=fail (critical)
    # Deps: curl, jq
    # Transport: Prometheus HTTP API at http://100.101.0.8:9090 (no SSH)
    ```

    Required structure (full implementation, not the abbreviated skeleton in RESEARCH.md):
    1. `set -euo pipefail`
    2. `PROM="${PROM_URL:-http://100.101.0.8:9090}"` (allow override)
    3. `CHECKED_AT=$(date -u +"%Y-%m-%dT%H:%M:%SZ")`
    4. `declare -A HOST_IP` with all 6 hosts mapped exactly per interfaces table
    5. `ALL_HOSTS=(tower docker-tower cc-worker mcow nether animaya-dev)`
    6. `prom_query()` helper — curl with `--data-urlencode "query=..."`, `--max-time 10`, `-sf`. On failure, print to stderr and return 1.
    7. `jq_value()` helper — extracts first result's `.value[1]` or returns empty string.
    8. `check_host()` — takes `host` arg, looks up IP, runs four checks, collects issues into an array, determines worst_exit (0/1/2), emits JSON via `printf` with proper escaping.
       - Check 1: up — worst_exit=2 on failure
       - Check 2: disk — worst_exit=2 if any mount ≤10% (query: `min by (instance) ((node_filesystem_avail_bytes{fstype!~"tmpfs|overlay|squashfs",instance="$ip:9100"} / node_filesystem_size_bytes{fstype!~"tmpfs|overlay|squashfs",instance="$ip:9100"}))`)
       - Check 3: memory — worst_exit=1 (degraded) if <10% available
       - Check 4: recent reboot — info only; add issue with metric:"uptime" value:"<Xs>" threshold:"180s" but do NOT change worst_exit
       - JSON emission: use `jq -c -n --arg host "$host" --arg status "$status" --arg ts "$CHECKED_AT" --argjson issues "$issues_json" '{host:$host,status:$status,issues:$issues,checked_at:$ts}'` — this guarantees valid JSON output. Avoid hand-rolled printf.
    9. `main()` parses args:
       - `--help` → print usage, exit 0
       - `--all` → iterate ALL_HOSTS, track max_exit, emit one JSON per line (NDJSON); final exit = max_exit
       - `<host>` → single check; if host not in HOST_IP map, print error and exit 2
       - no args → print usage, exit 2
    10. Stop conditions:
        - Prometheus unreachable (`prom_query` fails with curl timeout) → emit `{status: "fail", issues: [{metric: "prometheus", value: "unreachable", threshold: "http200"}]}` and exit 2. Do NOT let curl errors propagate as shell errors.
