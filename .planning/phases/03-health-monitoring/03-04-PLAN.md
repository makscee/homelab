    Two viable approaches:
    - **(a) Author by hand:** write the JSON directly using the Grafana dashboard schema (apiVersion 1, panels with type stat, targets using PromQL). Use these PromQL expressions (template variable `instance` one per host):
      - up: `up{job="node",instance="$instance"}`
      - CPU%: `100 - (avg by (instance) (rate(node_cpu_seconds_total{mode="idle",instance="$instance"}[5m])) * 100)`
      - RAM%: `(1 - (node_memory_MemAvailable_bytes{instance="$instance"} / node_memory_MemTotal_bytes{instance="$instance"})) * 100`
      - Disk%: `max by (instance) ((node_filesystem_size_bytes{fstype!~"tmpfs|overlay|squashfs",instance="$instance"} - node_filesystem_avail_bytes{fstype!~"tmpfs|overlay|squashfs",instance="$instance"}) / node_filesystem_size_bytes{fstype!~"tmpfs|overlay|squashfs",instance="$instance"} * 100)`
      - Uptime: `node_time_seconds{instance="$instance"} - node_boot_time_seconds{instance="$instance"}` (format: seconds (s))
    - **(b) Build in Grafana UI, export, normalize:** deploy datasource-only first, build dashboard in UI (`+ New dashboard`), `Share → Export → Save to file`, then apply the jq normalization from Task 1.
