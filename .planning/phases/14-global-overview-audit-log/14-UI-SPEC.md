# Phase 14 — UI Design Contract

> Visual and interaction contract for the `/` global overview page and the `/audit` read-only viewer. Generated by gsd-ui-researcher from Phase 13 carry-forward (slate + dark-default) and Phase 14 CONTEXT.md locked decisions (D-01 through D-17). Audit log infrastructure (SQLite + `logAudit()`) is backend-only and has no UI surface beyond `/audit`.

## Spacing Scale

Multiples of 4 — Tailwind defaults, carried verbatim from Phase 13. No custom extension.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`p-1`, `gap-1`) | Badge inner padding, stale-dot gap from label, inline icon gap |
| sm | 8px (`p-2`, `gap-2`) | Progress bar wrapper padding, load-avg triple spacing, card header→body gap |
| md | 16px (`p-4`, `gap-4`) | Card body padding, host-tile internal row rhythm, table cell padding |
| lg | 24px (`p-6`, `gap-6`) | Card outer padding, grid gap between host tiles, section padding |
| xl | 32px (`p-8`) | Page content container padding |
| 2xl | 48px (`p-12`) | Empty-state vertical padding (audit viewer with no rows) |
| 3xl | 64px | Not used this phase |

**Exceptions:**
- Host-tile sparkline column fixed at `h-10` (40px) × full card width inside a `px-4` inset — sparkline reads cleanly at this density without dominating the tile.
- Stale dot is a 8×8 absolute element (`h-2 w-2`) positioned via `gap-2` next to hostname — not a hit target, purely visual.
- Alerts-card severity chips use `h-6` (24px) badges stacked with `gap-2` to match Phase 13 badge rhythm.
- Audit table row min-height 48px (multiple of 4) — dense enough for scan, tall enough to keep truncated payload monospace legible.

## Typography

Slate preset inherits Tailwind defaults. Three sizes + one monospace exception — identical to Phase 13.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 (`leading-6`) |
| Label / table header / metric value | 14px (`text-sm`) | 600 (semibold) | 1.4 |
| Heading (page h1, card title) | 20px (`text-xl`) | 600 (semibold) | 1.3 |
| Display (big metric % in host tile) | 28px (`text-3xl`) | 600 (semibold) | 1.2 — **reserved for the three top metrics (CPU/mem/disk) on each host tile** |

**Monospace exception:** audit `action` column, `target` column, and `payload_json` preview use `font-mono text-xs` (12px) — alignment + "raw data" signal. Carried from Phase 13 rule for code-like content.

Uptime and load-avg triple are body size (14px) in `text-muted-foreground`, not display.

## Color

Inherits Phase 12 dark-default CSS variables (`apps/admin/app/globals.css`). Same 60/30/10 split as Phase 13 — no new tokens added this phase.

| Role | Value (HSL var) | Usage |
|------|-----------------|-------|
| Dominant (60%) | `--background` `222.2 84% 4.9%` | Page background, card gutters between tiles |
| Secondary (30%) | `--secondary` / `--muted` `217.2 32.6% 17.5%` | Host-tile card surfaces, alerts-card surface, audit table header, skeleton loaders, progress-bar track |
| Accent (10%) | `--primary` `210 40% 98%` | Primary sparkline stroke; progress-bar fill for CPU/mem/disk under their 80% threshold; Claude usage bar fill under 80%; `View all` link on AlertsCard; page-heading anchor focus ring |
| Destructive | `--destructive` `0 62.8% 30.6%` | `critical` severity alert badge; progress-bar fill for CPU/mem/disk ≥ 95%; red stale dot; Prometheus-outage `Alert` banner icon |

**Accent reserved for:**
1. Sparkline line stroke on each host tile (visual anchor for at-a-glance net I/O scan).
2. Progress-bar fill when value is `< 0.80` for CPU%, memory%, disk%, Claude 5h, and Claude 7d.
3. Link underlines / hover on "View all tokens" (→ `/tokens`), "View all alerts" (→ `/alerts`), and audit-row `target` when it resolves to a known resource.
4. Focus ring on all interactive elements (via `--ring`).

**Threshold color map (reused from Phase 13 — THIS IS THE SINGLE SOURCE):**
- `< 0.80` → accent / primary (safe / green semantic)
- `>= 0.80 < 0.95` → amber warning (`38 92% 50%`)
- `>= 0.95` → destructive

Applied uniformly to: CPU % bar, memory % bar, disk % bar, Claude 5h bar, Claude 7d bar.

**Stale dot color map (D-09, D-10) — re-uses the SAME three semantic colors:**
- `fresh` (age < 90s) → accent / primary (safe)
- `stale` (90s ≤ age < 300s) → amber warning
- `dead` (age ≥ 300s) → destructive
- `unknown` (no data) → `--muted-foreground` dimmed (neutral, not red — absence ≠ alarm)

**Alerts-card severity badge colors:**
- `critical` → destructive
- `warning` → amber
- `info` / other → `--secondary` (neutral)
- `0 firing` across all severities → single neutral badge "All clear" using `--secondary`

No color drift beyond what is listed here. Amber warning is the SAME token declared in Phase 13 UI-SPEC; reused, not re-declared.

## Interaction Contract

### Overview page — refresh cadence

- Initial render is RSC-populated (seed fetch at request time from `/api/overview` logic inline, OR a fetch in the server component).
- Client attaches SWR with `refreshInterval: 30_000`, `revalidateOnFocus: false`. One hook per logical endpoint: `useOverview()` on `/api/overview`, `useAlertCount()` on `/api/alerts/count`.
- During the 30s interval the UI holds last-known values; a stale dot communicates freshness at the tile level.
- Tab-visibility pause is handled by SWR default (do not override).

### Host tile

- Pure visual card. No interaction beyond hover (subtle `--secondary` background lift) and tooltip on the stale dot.
- Not clickable — there is no `/hosts/[name]` route in this phase (per CONTEXT.md deferred). A future phase can make the card a link without breaking this contract.
- Progress bars expose `role="progressbar"` with `aria-valuenow`, `aria-valuemin=0`, `aria-valuemax=100`, and an `aria-label` ending in the raw percentage (e.g. `"CPU: 47%"`). Color is never the only threshold signal — numeric `{pct}%` renders to the right of each bar track.

### Claude usage summary card

- Each per-token mini card is a link-surface: the whole card navigates to `/tokens/[id]`. Card header icon `lucide-react` `ExternalLink` hints at navigation.
- Bars use the same color threshold as Phase 13 token list. Do not re-derive — reuse the `thresholdClass()` helper from `app/(auth)/_lib/view-model.ts`.

### Alerts card

- Card is a link-surface to `/alerts` (stub route until Phase 17 wires the real page).
- Severity breakdown list rendered only when `total > 0`. When `total === 0`, only the `All clear` body is rendered.
- When the `ALERTS{}` query returns `[]` (no rules loaded yet pre-Phase 17), treat as `0 firing` — NOT as an error. The tooltip surfaces the no-rules nuance for operators.

### Alert badge in nav

- Rendered inside the top bar layout, right-aligned next to the user avatar.
- When `firingCount === 0`, the badge is hidden entirely (not "0"). This keeps the nav quiet when the homelab is healthy.
- Count updates via the same SWR poll as the overview card — a shared `useAlertCount()` hook ensures both surfaces agree.

### Degraded mode — per-host vs. full outage (D-10)

- Per-host query failure: tile renders with last-known values + red stale dot. No page-level banner.
- Prometheus HTTP call itself failed: page-level destructive `Alert` banner appears above the grid. Tiles retain last-known values; stale dots go red across the board. No form controls are disabled (there are no forms on this page).

### `/audit` — pagination (D-17)

- Cursor: `?before=<id>`. Default page size 50.
- `Older →` link sets `?before={lastIdOnPage}`.
- `Newer` link removes `?before=` param (snaps to first page). Hidden when already on first page.
- No filters, no search, no sort — all deferred to Phase 19.

### `/audit` — payload cell

- Row click on the `Payload` cell toggles expand/collapse. No modal.
- When expanded, the row height grows; other rows are unaffected.
- Payload JSON is already server-side-redacted before it is stored (deny-list + token pattern); the UI MUST NOT re-redact. The client renders raw DB content.
- If `payload_json` is `NULL`, render `—` in `text-muted-foreground` (not clickable).

### Keyboard + focus

- Tab order on `/`: Overview heading anchor → each host tile (inert, skipped if non-interactive) → each Claude mini card (link) → Alerts card (link) → nav chrome.
- Tab order on `/audit`: table header (non-interactive) → each row's expand toggle → `Newer` / `Older` buttons.
- `Esc` has no handler (no modals open on these pages).
- Focus ring uses `--ring` globally — no per-component override.

### Toasts

- No toasts in this phase. The overview is read-only; the audit viewer is read-only. Error states surface via inline banners and page-level `error.tsx`, not toasts.

## Accessibility

- Progress bars: `role="progressbar"` + `aria-valuenow`/`min`/`max` + `aria-label` with raw percentage (`"CPU: 47%"`, `"5h usage: 62%"`).
- Stale dot: `aria-label="status: fresh|stale|dead|unknown"` on the dot element so screen readers announce staleness without relying on color.
- Severity color is never the sole signal on the Alerts card — each row is `"{severity_name} · {count}"` as readable text.
- Relative timestamps on `/audit` MUST carry an `aria-label` with the absolute ISO timestamp (e.g. `aria-label="2026-04-15T14:23:01Z"`) so screen readers don't lose resolution.
- Sparkline is decorative; the accompanying `sr-only` caption carries peak throughput so the chart is non-essential for understanding.
- Nav alert badge carries `aria-live="polite"` so count changes are announced when they cross zero.
- No destructive action on these pages → no confirmation ceremony required.
