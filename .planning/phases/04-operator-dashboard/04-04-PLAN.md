    **Step A — Locate operator pubkey.** On operator machine:
    ```bash
    PUBKEY=$(cat ~/.ssh/id_ed25519.pub 2>/dev/null || ssh root@tower 'cat /root/.ssh/authorized_keys' | grep 'ssh-ed25519' | head -1)
    echo "$PUBKEY"
    # Sanity: must start with 'ssh-ed25519 AAAA'
    echo "$PUBKEY" | grep -qE '^ssh-ed25519 AAAA[A-Za-z0-9+/=]{60,}'
    ```
    If empty or malformed, STOP — ask user which key to use.

    **Step B — Verify pct exec 205 works from tower.** (LXC 205 is animaya-dev.)
    ```bash
    ssh root@tower 'pct list | grep -E "^205\s"'
    # expect: "205 running animaya-dev" or similar
    ssh root@tower 'pct exec 205 -- hostname'
    # expect: something containing "animaya" — proves pct exec works
    ```
    Stop condition: LXC 205 not found, not running, or `pct exec` returns non-zero.

    **Step C — Create .ssh dir + authorized_keys with correct perms inside LXC.**
    ```bash
    # Use heredoc via ssh → pct exec to inject the key safely
    ssh root@tower "pct exec 205 -- mkdir -p /root/.ssh"
    ssh root@tower "pct exec 205 -- chmod 700 /root/.ssh"
    ssh root@tower "pct exec 205 -- chown root:root /root/.ssh"

    # Append the key idempotently (skip if already present)
    ssh root@tower "pct exec 205 -- bash -c 'grep -qxF \"$PUBKEY\" /root/.ssh/authorized_keys 2>/dev/null || echo \"$PUBKEY\" >> /root/.ssh/authorized_keys'"
    ssh root@tower "pct exec 205 -- chmod 600 /root/.ssh/authorized_keys"
    ssh root@tower "pct exec 205 -- chown root:root /root/.ssh/authorized_keys"
    ```

    **Step D — Verify perms + key presence inside LXC.**
    ```bash
    ssh root@tower 'pct exec 205 -- stat -c "%a %U:%G" /root/.ssh'
    # expect: 700 root:root
    ssh root@tower 'pct exec 205 -- stat -c "%a %U:%G" /root/.ssh/authorized_keys'
    # expect: 600 root:root
    ssh root@tower "pct exec 205 -- grep -c '$(echo $PUBKEY | awk '{print $2}' | head -c 40)' /root/.ssh/authorized_keys"
    # expect: >= 1
    ```

    **Step E — Verify sshd config allows root publickey login.**
    ```bash
    ssh root@tower 'pct exec 205 -- grep -E "^(PermitRootLogin|PubkeyAuthentication)" /etc/ssh/sshd_config'
    # expect: PermitRootLogin yes (or prohibit-password), PubkeyAuthentication yes
    # If either forbids or missing → set it:
    ssh root@tower 'pct exec 205 -- bash -c "sed -i \"s/^#*PermitRootLogin.*/PermitRootLogin prohibit-password/\" /etc/ssh/sshd_config &amp;&amp; sed -i \"s/^#*PubkeyAuthentication.*/PubkeyAuthentication yes/\" /etc/ssh/sshd_config &amp;&amp; systemctl reload sshd"'
    ```

    **Step F — End-to-end SSH test from operator.**
    ```bash

    ssh-keygen -R 100.119.15.122 2>/dev/null
    # Accept new host key on first connect
    ssh -o StrictHostKeyChecking=accept-new -o BatchMode=yes root@animaya-dev 'hostname &amp;&amp; uname -a'
    # expect: hostname output + uname, no password prompt, exit 0
    ```
    Stop condition: Still prompts for password → sshd config or key mismatch.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Operator → tower → LXC 205 | Proxmox host trust chain (SSH to hypervisor, pct exec into container) |
| pct exec → authorized_keys write | Key material is a PUBLIC key — non-secret |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04-01 | Information Disclosure | Private SSH key leakage via pct exec | mitigate | Only the PUBLIC key (`~/.ssh/id_ed25519.pub`) is pushed. The pubkey sanity check requires `ssh-ed25519 AAAA...` prefix. The action never reads, logs, or transmits the private key. |
| T-04-04-02 | Elevation of Privilege | Weak permissions on authorized_keys | mitigate | Explicit `chmod 700` on `.ssh/` and `chmod 600` on `authorized_keys`; acceptance criteria greps the stat output for these exact modes. sshd refuses loose perms. |
| T-04-04-03 | Spoofing | MitM on first SSH connection accepting wrong host key | accept | Initial `accept-new` is acceptable: connection is over Tailnet to a specific LXC IP (100.119.15.122) — Tailscale provides authenticated routing. Future connections will fail if host key changes. |
| T-04-04-04 | Tampering | Malicious key injection elsewhere in authorized_keys | mitigate | Idempotent append (`grep -qxF` guard) prevents duplicate; Step D verifies the specific pubkey fingerprint is present. Evidence log commits the fingerprint. |
| T-04-04-05 | Denial of Service | sshd reload disrupts existing connections | accept | `systemctl reload sshd` (not restart) preserves established sessions; only applied if config needed changes. |

</threat_model>
